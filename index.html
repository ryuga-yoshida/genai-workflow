生成AIを活用した開発ワークフロー

アイデア創出から設計・実装・運用まで、工程ごとに役割の異なる生成AIを使い分けることで、開発の質とスピードを両立させる取り組みです。実際に開発・運用している就活支援アプリ（Web／iOS）を活用事例に、その進め方を紹介します。

アイデア出し ── ChatGPT プロトタイプ ── Manus 設計相談 ── Claude コーディング ── Claude Code
01 ｜ 目的

1つのAIに任せきらず、工程ごとに使い分ける

同じ「生成AI」でも、発想を広げることに向くもの、動くものを素早く形にするもの、論理的に設計を詰めるもの、実際にコードを記述するものがあります。それぞれの得意分野に合わせて工程ごとに使い分けることで、検討の抜け漏れを抑えつつ開発を効率化しています。

POINT 01

検討段階から相談する

企画・設計の段階からAIを相談相手とし、論点の洗い出しと検討の質を高めます。

POINT 02

早期に動かして確認する

アイデアを試作で形にし、本格的に作り込む前に方向性を確認します。

POINT 03

実装・運用までつなげる

設計から実装、本番運用までを一連の流れとして進めます。

02 ｜ 全体像

4つの工程を、リレー形式で進める

各工程の成果を次の工程へ受け渡しながら、企画から実装まで一貫して進めます。

03 ｜ 工程別の活用ツール

各工程で、何にどう使うか

STEP 01 ｜ アイデア出し
ChatGPT
企画の発散・ブレスト
用途
企画の発散、想定課題の洗い出し、機能の優先度づけ、表現案の作成。
選定理由
幅広い案を返し、対話を重ねて発想を広げる用途に適している。
具体例
「対象ユーザーがつまずく場面」を起点に機能案を洗い出し、優先度を整理する。
STEP 02 ｜ プロトタイプ
Manus
試作による検証
用途
アイデアを操作できる試作にし、画面遷移やデータの流れを確認する。
選定理由
指示から動く試作を生成でき、作り込む前に体験で確認できるため手戻りを抑えられる。
具体例
主要画面の試作を作成し、使い勝手を見極めてから本実装に移る。
STEP 03 ｜ 設計相談
Claude
設計・技術選定の検討
用途
構成、技術選定、データモデル、各案のトレードオフの検討。
選定理由
長い文脈を保ったまま、判断の根拠を示しながら整理できる点が設計検討に適している。
具体例
「この構成で妥当か、代替案はあるか」を相談し、データ設計や方針を固める。
STEP 04 ｜ コーディング
Claude Code
実装・修正・デプロイ
用途
既存のコードベース上での実装、リファクタリング、テスト、デプロイ。
選定理由
複数ファイルを横断して読み書きし、コマンド実行まで行える開発支援ツールである。
具体例
機能追加からクラウドへのデプロイ、不具合修正までを一連の流れで実行する。
04 ｜ 外部サービスとの連携

生成AIを組み込み、クラウドで運用する

生成AIで開発するだけでなく、外部サービスと連携して本番運用までつなげています。AIを製品の機能として組み込み、クラウド上で公開し、コードの変更が自動で反映される構成です。

Gemini
AI機能の実装

文章添削やフィードバック、企業情報の生成など、アプリ内のAI機能をGemini APIで実装。生成AIを製品の機能として組み込んでいます。

Google Cloud
本番インフラ

Cloud Run でアプリを公開し、Cloud Build で自動デプロイ。データベースやストレージ、定期処理もクラウド上で運用しています。

GitHub
ソース管理・デプロイ起点

ソースコードをバージョン管理し、開発から本番への反映で自動ビルド・デプロイが実行されます。Claude Code と連携して運用しています。

連携の流れ ：　Claude Code　→　GitHub（コード管理）　→　Google Cloud（自動デプロイ）　／　アプリは Gemini で動作
05 ｜ 業務への応用

社内で使えるAIを前提に、業務へどう活かすか

業務では、用途に応じて複数の生成AIを活用できます。例えば、エディタ上で使う GitHub Copilot（VS Code） や、ターミナルから使う Claude Code、社内データにも接続でき機密も扱える 社内チャットAI などです。日々の業務にはドキュメント作成・既存コードの調査・テスト・定型作業といった地道な工程が多く含まれ、こうした作業こそAIを活かして着実に効率化できる領域です。ツールごとの強みに合わせて使い分けることで、その効果はさらに高まります。

社内チャットAI機密対応
社内データに繋いで、安全に・幅広く
  • 社内文書やWebに接続して要約・検索（機密情報も入力可）
  • 議事録づくりや文章の要約・整形など業務向けの機能が豊富
  • 用途に応じてモデル（GPT-5.1／Gemini 2.5）を切替
機密を扱え、開発以外の業務も広くカバーする中心的な入口
GitHub CopilotVS Code
エディタの中で、書きながら使う
  • コーディング中の補完・候補提示でテンポを上げる
  • エディタ内での質問・コードの説明・調査
  • その場で完結する編集（差分を見て採用/却下）
手を動かしている最中の作業を、隣で補助する
Claude CodeAPI経由・CLI
ターミナルで、まとめて任せる
  • 既存コードの横断調査・複数ファイルの改修
  • まとまった実装・修正を一連で実行
  • 定型処理のスクリプト化・自動化
一連の作業を渡して、自走で進めてもらう

※ とくに GitHub Copilot と Claude Code はできることが重なる部分も多く、明確に分かれるわけではありません。「エディタ内で書きながら使う」か「まとまった作業を任せる」か、場面と強みに合わせて使い分けます。また、初期の発散や雑多な相談はチャットAIで進めると、コーディングAI（Claude Code）の限られた文脈（コンテキスト）を実装作業に集中させられます。

01

設計書・ドキュメント作成

社内チャットAIで仕様書・報告書・議事録の下書きや既存資料の要約を作り、人は内容の確認と調整に集中します。

02

テスト・調査作業の補助

テスト観点やテストケースの洗い出し、ログ・エラーメッセージの調査をCopilotで補助します。

03

既存コードの理解・改修

Claude Code（CLI）で大きな既存コードを読み解き、修正箇所の特定や改修のたたき台づくりに活用します。

04

繰り返し作業の自動化

データ整形や一括変換などの定型作業をClaude Codeでスクリプト化し、手作業を削減します。

作業例：既存システムの改修を1件こなす流れ
1
社内チャットAI

調査・方針づくり

仕様書や過去資料・社内情報を検索して背景を把握し、改修方針のたたき台をまとめる。

2
Claude Code（CLI）

コード調査・実装

既存コードを横断的に読み解いて修正箇所を特定し、改修のたたき台を実装する。

3
GitHub Copilot（VS Code）

仕上げ

エディタ内で補完を効かせながら細部を調整し、コードを仕上げる。

4
社内チャットAI Claude Code / Copilot

テスト・報告

テスト観点の洗い出しや報告書・エビデンスの文面づくりは社内チャットAIで行い、テストケースや報告書は Claude Code・Copilot で Excel 等のファイルとして出力する（スクリプト生成で自動化）。

留意点　モデルが最新版でなくても、用途を絞れば実用上は十分です。また、機密情報・個人情報の入力は社内ルールに従い、利用可否を確認したうえで取り扱います。AIの出力は最終確認を前提とし、判断と責任は人が担います。
まとめ

生成AIを「役割で使い分ける」ことで、アイデアから実装・運用までを一連の流れとして進められます。この考え方は、社内で使えるAI（GitHub Copilot・Claude Code）の範囲でも、企画・調査・設計・実装の各業務にそのまま応用できると考えています。
